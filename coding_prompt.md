# 项目沟通与执行 Prompt（约定版）

目的：固化我们的协作模式，保证需求确认、过程记录与复盘一致性。

## 核心约定
1. 先确认项目细节  
   - 在启动任何构建/实现前，必须完成项目细节确认。  
   - 未完成确认时，只允许澄清问题与方案讨论，不得创建或修改项目文件。
2. 全程记录到 Markdown  
   - 所有新增、修改、变动的内容，都必须同步记录到指定的 `md` 文档中。  
   - 该文档作为项目过程日志，便于项目完成后复盘。
3. 中文沟通与输出  
   - 所有沟通、思考、输出内容均使用中文。
4. 统一使用 `.env` 管理可配置项  
   - 所有环境变量、可调整参数与密钥，统一写入 `.env` 文件。  
   - 非必要配置不应硬编码在代码或配置文件中。
   - `.env` 默认不上传到 GitHub
5. 默认使用 conda 与 Python  
   - 未特殊说明时，默认使用 conda 创建 Python 虚拟环境。  
   - 后续生成的代码默认使用 Python 语言。

## 执行顺序（必须遵守）
1. 收集与确认项目细节  
2. 汇总确认结果并征得明确开始指令  
3. 开始搭建项目（按约定记录变更）

## 变更记录要求
- 每次新增/修改/变动，都要同步到项目的记录型 `md` 文档中。  
- 记录应包含：变更内容、变更原因、相关文件路径（若适用）。
